# compound-interest-calculator

## Context (зачем это)

Простая интерактивная страница, где пользователь прикидывает, сколько накопится
за N лет с учётом начального капитала, регулярных пополнений и капитализации
процентов. Наглядный инструмент для планирования личных накоплений.

### Ключевые решения

- **Стек:** Vue 3 + Tailwind CSS, обычный **JavaScript** (без TypeScript).
- **Скрипт** написан в стиле **Composition API** Vue 3 (`createApp({ setup() {...} })`,
  `ref`, `computed`).
- **Без сборки и без локальных серверов** (никакого Node.js, Vite, http-server).
  Это одна веб-страница, которая открывается прямо в браузере (двойной клик по
  файлу). Vue и Tailwind подключаются через CDN.
- **Результат:** таблица с разбивкой по годам (итоговая накопленная сумма видна
  как «Баланс на конец года» в последней строке). Таблица на странице **всегда**;
  до первого расчёта показаны только её заголовки (без строк).
- **Валюта:** рубли (₽).
- **Регулярность пополнений** и **Капитализация процентов** — выбор через
  **radio-кнопки**.
- **Без дефолтных значений** у числовых полей. Тексты ошибок по умолчанию скрыты
  и появляются только после нажатия кнопки «Рассчитать».
- **Заголовок:** «Калькулятор сложного процента» (без пояснительного текста).

### Стилизация — Tailwind-утилитами

Все стили — **Tailwind-классами прямо в разметке** (без отдельного авторского
`<style>` с CSS-классами). Точные значения задаются произвольными утилитами
(`w-[170px]`, `border-[0.5px]`, `bg-[rgba(128,128,128,0.22)]` и т.п.).

Соответствие брейкпоинтов (mobile-first):
- **база** (без префикса) — мобильная версия;
- **`sm:`** (= `min-width: 640px`) — десктоп.

Чётные строки таблицы — арбитрарный вариант на `<tbody>`
(`[&_tr:nth-child(even)_td]:bg-[rgba(128,128,128,0.10)]`). Скрытие полосы
прокрутки у таблицы Tailwind из коробки не умеет — для него добавлена **кастомная
утилита `.scrollbar-hide`** в блоке `<style type="text/tailwindcss">` (`@layer
utilities`).

## Поля ввода (реактивное состояние, Composition API)

Дефолтов у числовых полей нет — стартуют пустыми (`ref('')`). Плейсхолдеров в
полях нет. Radio-группы имеют начальный выбор.

| Подпись поля | Переменная | Контрол | Значения | Старт |
|------|-----------|---------|----------|-------|
| Начальная сумма | `initial` | `input[type=text]` | число ≥ 0 | `''` |
| Сумма пополнения | `contribution` | `input[type=text]` | число ≥ 0 | `''` |
| Пополнения | `contributionFreq` | **radio** | `monthly` / `yearly` | `monthly` |
| Ставка, % | `ratePercent` | `input[type=text]` | число ≥ 0 | `''` |
| Капитализация | `compounding` | **radio** | `semiannual` / `yearly` | `yearly` |
| Срок, лет | `years` | `input[type=text]` | целое ≥ 1 | `''` |

Числовые поля — `input[type=text]` (с `inputmode`), потому что в них показывается
**форматированное** значение (см. ниже), а `type=number` такого не позволяет.
Подписи числовых полей — без символа валюты (₽). В состоянии (`ref`) хранится
«сырое» числовое значение (только цифры и, для сумм/ставки, десятичная точка); всё
форматирование — только на отображение.

### Форматирование значений в полях

Число в поле показывается в том же формате, что и в таблице, но **только когда
поле не в фокусе**. При фокусе показывается «сырое» значение (одни цифры) — так
удобно редактировать; при потере фокуса (`@blur`) значение снова форматируется.

- **Начальная сумма, Сумма пополнения** — денежный формат, как в таблице
  (`100 000,00 ₽`).
- **Ставка** — число с двумя знаками после запятой, без `₽` (`10,00`).
- **Срок** — целое число, без знаков после запятой (`5`).

### Запрет ввода некорректных значений (на уровне поля)

- **Все числовые поля** — нельзя ввести отрицательное значение. В `@keydown`
  блокируем символы `-`, `+`, `e`; в `@input` санитизируем значение (оставляем
  только цифры и — для сумм и ставки — одну десятичную точку, запятую приводим к
  точке).
- **Срок** — только целое число: в `@input` оставляем одни цифры (никаких
  разделителей дробной части).

### Валидация (показывается только после «Рассчитать»)

`computed`-объект `errors` — по ключу на каждое числовое поле, значение = текст
ошибки или `null`:
- пусто → «Заполните поле»;
- `years`: не целое или `< 1` → «Введите целое число от 1».

Флаг `showErrors` (`ref(false)`) — изначально `false`, поэтому ошибки **не
видны**. Нажатие «Рассчитать» ставит `showErrors = true`. Текст ошибки рендерится
под полем при `showErrors && errors.<поле>`; у невалидного поля граница
становится красной и **чуть тоньше** по ширине, чем обычная (тонкая красная
рамка, без толстой подсветки/свечения). Размер шрифта ошибок — **чуть меньше**
меток/текста полей (метки `text-sm` → ошибки `text-xs`).

`computed isValid` = все ошибки `null`.

## Поведение кнопки «Рассчитать» и пересчёт

Таблица пересчитывается **только по нажатию кнопки** (не реактивно «на лету»).
Дополнительно расчёт запускается по нажатию **Enter** в любом числовом поле —
эффект тот же, что и у кнопки (вызывает `calculate()`).

- Состояние результата хранится в `ref` `rows` (изначально `[]`).
- Метод `calculate()`:
  1. `showErrors = true`;
  2. если `isValid` — посчитать и записать строки в `rows`;
  3. если невалидно — `rows` не обновляется (под полями видны ошибки).
- Таблица в разметке **всегда** (заголовки видны всегда); тело таблицы — это
  `v-for` по `rows`, поэтому до первого успешного расчёта строк нет (пустая
  таблица с заголовками). После нажатия ошибки обновляются реактивно по мере
  исправления полей, но **строки меняются только при следующем нажатии** кнопки.

## Логика расчёта (помесячная симуляция)

Считаем по месяцам (12 шагов в году) — в 12 месяцев ровно укладываются все
комбинации частот, одна петля покрывает всё.

Параметры внутри:
- `k` = капитализаций в год: `semiannual → 2`, `yearly → 1`;
- ставка за период капитализации = `ratePercent/100 / k`;
- месяцы капитализации: `k=2` → месяцы 6 и 12; `k=1` → месяц 12;
- месяцы пополнения: `monthly` → каждый месяц; `yearly` → месяц 12.

Псевдокод:

```
balance = initial
rows = []
for year in 1..years:
    startBalance = balance
    interestThisYear = 0; contribThisYear = 0
    for month in 1..12:
        if isContributionMonth(month):
            balance += contribution; contribThisYear += contribution
        if isCompoundingMonth(month):
            interest = balance * periodRate
            balance += interest; interestThisYear += interest
    # Баланс до начисления процентов за год = остаток прошлого года + пополнения года
    baseBeforeInterest = startBalance + contribThisYear
    # Значения для строки округляем до сотых (2 знака)
    rows.push({ year,
                contribThisYear: round2(contribThisYear),
                baseBeforeInterest: round2(baseBeforeInterest),
                interestThisYear: round2(interestThisYear),
                balance: round2(balance) })
return rows
```

`round2(x) = Math.round(x * 100) / 100`. Внутренний `balance` переносится между
годами с полной точностью; округление до сотых применяется только к значениям,
попадающим в строку (на вывод).

Расчёт вызывается из `calculate()`.

**Соглашение по порядку операций** (отразить в комментарии): в течение месяца
сначала вносится пополнение, затем (в месяц капитализации) начисляется процент на
текущий баланс. Простая и предсказуемая модель.

## Разметка / UI (Tailwind)

Фон всей страницы — **мягкий светлый тёплый серый** (`#F4F3EF`): спокойный,
нейтральный, не бликует и не перетягивает внимание с содержимого/таблицы. Контент
страницы — с внутренним отступом от краёв документа (padding на корневом
контейнере): **12px на мобильной версии** и **20px на десктопе**
(`@media (min-width: 640px)`). На мобильной версии отступ между всеми элементами
панели (полями, radio-группами и кнопкой) — **одинаковый, 8px**.

Сверху — заголовок, под ним панель управления, ещё ниже — таблица результатов.

1. **Заголовок** — `<h1>Калькулятор сложного процента</h1>`, без подзаголовка.
   Размер/жирность задаём явными Tailwind-классами `text-2xl font-bold` (24px / 700);
   это нужно, потому что Tailwind-preflight
   сбрасывает заголовки до размера обычного текста. Нижний отступ до первого поля —
   **10px на мобильной версии** и **20px на десктопе** (`@media (min-width: 640px)`).
2. **Панель управления** — **две группы в одной переносимой строке** (внешний
   контейнер `display: flex; flex-wrap: wrap; align-items: flex-start`, промежуток
   между группами **24px**). На десктопе группы стоят рядом; при сужении экрана
   вторая группа **переносится под первую**. **Фона у ячеек нет** (плиток нет).
   - **Группа полей** — сетка **2×2** (`grid`, промежуток **16px**): 4 числовых
     поля (1-я строка — начальная сумма и пополнение, 2-я — ставка и срок).
   - **Группа «radio + кнопка»** — `flex; flex-wrap: wrap; align-items: flex-start;
     gap: 24px`: блок из двух radio-групп («Пополнения» и
     «Капитализация») — сам тоже `flex; flex-wrap: wrap` (его `gap` **не меняется**,
     всегда 24px), и кнопка «Рассчитать».
     `flex-wrap: wrap` — при нехватке места кнопка переносится под radio-группы.
     На десктопе при переносе всей группы «radio + кнопка» под группу полей она
     **растягивается на всю ширину страницы** (`flex-grow: 1`).
   - **Выравнивание** — всё **по верхнему краю**: и две группы (поля и
     «radio + кнопка») на уровне панели, и кнопка внутри своей группы
     (`align-items: flex-start` на всех уровнях).
   - **Числовое поле** — `<input type="text">` высотой **40px**, с `id` и
     `<label for>`, **без плейсхолдера**; между подписью и полем — **4px**; **без
     нижнего отступа** (текст ошибки не резервирует пустое место — появляется
     только при ошибке). Ширина поля сумм («Начальная сумма», «Сумма пополнения»)
     — **170px** (вмещает форматированное число до триллиона —
     `1 000 000 000 000,00 ₽`). Поля «Ставка, %» и «Срок, лет» — **узкие**
     (`w-16`, под ~3 цифры), т.к. там небольшие значения. На мобильных высота поля
     — **36px** (см. «Адаптивность»).
   - **Radio-группа** — подпись и под ней radio **один под другим**
     (`<input type="radio">` + подпись, `v-model`); между подписью и опциями — 4px.
   - **Подписи полей и radio-групп — жирные** (`font-weight: 700`).
   - **Кнопка «Рассчитать»** — высота **40px**, **чёрного** фона, текст — **белый**.
3. **Таблица результатов по годам** (снизу): столбцы
   `Год | Пополнения | Пополнения за год | Проценты | Итого`, где
   **«Пополнения за год»** — баланс до начисления процентов за год (остаток с
   прошлого года + пополнения года; равен «Итого» − «Проценты»), а **«Итого»** —
   баланс на конец года. Сама
   таблица с заголовками присутствует всегда; строки тела — `v-for="row in rows"`
   (до первого расчёта тело пустое, видны только заголовки). Заголовки выделены
   **серым фоном** (умеренно-серый, `rgba(128,128,128,0.22)`), причём каждая
   ячейка-заголовок — отдельный «чип» со скруглением **8px**; **промежуток между
   колонками таблицы — 4px** (`border-spacing`). Заголовки **жирные**, причём
   `font-bold` задаётся на уровне строки `<tr>` (а не на каждом `<th>`) и
   наследуется ячейками. **Чётные строки тела** подсвечены
   более светлым серым (`rgba(128,128,128,0.10)`, светлее заголовков). **Значения
   во всех колонках выровнены по левому краю** (и заголовки, и ячейки тела).
   Ширина таблицы — **на всю ширину** (`width: 100%`). На **мобильной версии**
   обёртка таблицы вытянута до **правого края страницы** (отрицательный
   `margin-right`, гасящий правый паддинг страницы) — так таблица заметно «уходит»
   за край и видно, что её можно прокрутить вправо.

Форматирование сумм — `Intl.NumberFormat('ru-RU', { style:'currency',
currency:'RUB', minimumFractionDigits:2, maximumFractionDigits:2 })`, метод
`fmt(value)`. Числовые значения округляются **до сотых** (2 знака после запятой).

### Адаптивность

Подход — **mobile-first** (как в Tailwind): базовые стили описывают мобильную
версию, а десктоп включается медиа-запросом **`@media (min-width: 640px)`** (это
граница `sm` Tailwind). Так не возникает «нахлёста» правил на самой границе.

Главный принцип: **страница не должна переполняться по горизонтали** — иначе
мобильный браузер уменьшает её целиком (zoom-out) и всё становится мелким.

**Базовый уровень — мобильная версия** — всё в **один столбец**:
- панель и группа полей — в один столбец; **блок radio-групп — `flex-direction:
  row` всегда** (группы в ряд, переносятся при нехватке места);
- **поля и кнопка — фиксированной ширины 170px на всех экранах** (без
  растягивания на всю ширину); сетка полей — `grid-cols-[repeat(auto-fit,170px)]`,
  поэтому 170px-поля сами переносятся по мере ширины (на узких экранах — по одному
  в ряд);
- отступы между элементами — **8px**; паддинг страницы — **12px**; высота
  числовых полей — **36px**;
- **таблица — в обёртке с горизонтальной прокруткой** (`overflow-x: auto`),
  значения в ячейках **не переносятся** (`white-space: nowrap`). **Полоса
  прокрутки скрыта** (`scrollbar-width: none`; `-ms-overflow-style: none`;
  `::-webkit-scrollbar { display: none }`) — скролл вбок работает, но визуально
  полосы нет. На мобильной версии обёртка **доходит до правого края страницы**
  (`margin-right: -12px`), показывая, что таблица уходит за экран; `padding-right:
  12px` даёт при **полной прокрутке вправо** такой же отступ от края, как слева
  (12px). На десктопе (≥640px) сдвиг и паддинг сбрасываются (`margin-right: 0;
  padding-right: 0`).

**Десктоп (`@media (min-width: 640px)`)**:
- группа полей (2×2) и группа «radio + кнопка» — в одной строке (внешний
  `flex`), выровнены по верхнему краю; когда группа «radio + кнопка» не помещается
  и переносится под группу полей, она **растягивается на всю ширину**
  (`flex-grow: 1`);
- поля — высота **40px**; отступы — **16px** в сетке полей и **24px** между двумя
  группами; `gap` у блока radio-групп не меняется (24px); паддинг страницы —
  **20px**.

## Verification (как проверять при реализации)

1. Открыть `compound-interest-calculator.html` двойным кликом в браузере —
   страница должна работать без сервера.
2. Консоль браузера чистая, Vue-приложение смонтировалось.
3. На старте: ошибок под полями нет; таблица видна, но пустая (только заголовки).
4. Нельзя ввести отрицательное значение в «Сумма пополнения», «Ставка, %»,
   «Срок» (и «Начальная сумма»); в «Срок» нельзя ввести дробное.
5. Нажатие «Рассчитать» при пустых полях → под полями появляются сообщения об
   ошибках; таблица не строится.
6. Контрольные значения (после «Рассчитать»):
   - 100 000 ₽, пополнение 0, ставка 10%, капитализация ежегодная, 1 год →
     баланс года 1 = **110 000,00 ₽**.
   - 100 000 ₽, пополнение 0, ставка 10%, капитализация раз в полгода, 1 год →
     `100000·1.05·1.05 =` **110 250,00 ₽**.
7. Пересчёт происходит только по кнопке: изменение поля без нажатия не меняет
   таблицу.
8. Адаптивность: сужаем окно — поля и таблица сужаются, контент не ломается.

## Вне рамок (не делаем)

- TypeScript, любые сборщики (Vite) и локальные серверы (Node.js, http-server).
- График (Chart.js); отдельный блок-сводка («Итого/Вложено/Доход») — оставляем
  только таблицу.
- Выбор валюты (только ₽), инфляция, налоги, частичные снятия, деплой.
